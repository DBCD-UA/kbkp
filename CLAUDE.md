## Architecture

This is a **Next.js 15 + Payload CMS 3 + PostgreSQL** full-stack application for a Ukrainian electronics manufacturing company (КБКП).

### Route Groups

The `src/app/` directory uses two route groups:

- `(frontend)/` — Public-facing website: home page, product catalog, about page
- `(payload)/` — Payload CMS admin panel at `/admin` and REST/GraphQL API at `/api`

### CMS Collections (`src/collections/`)

Payload CMS collections define the data model:

- **Products** — Title, category (automation/telecom/energy), rich text description (Lexical editor), image gallery, features list
- **Users** — Auth-enabled user accounts for CMS access
- **Media** — File uploads used across collections

Types are auto-generated into `src/payload-types.ts` — do not edit this file manually. Re-run `npm run build` or restart the dev server to regenerate after collection changes.

### Key Files

- `src/payload.config.ts` — Payload CMS configuration (collections, database, plugins)
- `src/app/(frontend)/layout.tsx` — Root frontend layout
- `src/components/` — Shared React components (Header, Footer, Hero, ContactModal)

### Tech Stack

- **Next.js 15** with App Router and React 19 + React Compiler
- **Payload CMS 3** (headless, self-hosted) with Lexical rich text editor
- **PostgreSQL** via `@payloadcms/db-postgres` — connection string in `DATABASE_URL` env var
- **Tailwind CSS v4** via PostCSS
- **TypeScript** with strict mode; path alias `@/*` → `src/*`

## Known Issues

`next.config.ts` has `withPayload` redundantly nested 5 times — it should be `withPayload(nextConfig)` once.

## Environment

Requires a `.env` file with:
- `DATABASE_URL` — PostgreSQL connection string
- `PAYLOAD_SECRET` — Secret key for Payload CMS authentication
